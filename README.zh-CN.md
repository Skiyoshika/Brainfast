# Brainfast

**面向增强子 AAV 全脑光片数据的可重复、GUI 驱动全脑分析流程。**

> **致答辩委员会：** Brainfast 不是一个新算法——它是一项 *工作流程与分析基础设施* 贡献。它的目标是将实验室碎片化、多工具、高度依赖人工的分析流程整合为一条可重复的 pipeline，产出结构化的生物学数据。科研价值在于可重复性、步骤压缩与定量区域输出，而非在算法上超越现有单一工具。

---

## 动机 — 为什么做 Brainfast？

### 现有实验室流程的问题

全脑光片荧光显微成像后的增强子 AAV 表达定量，涉及一系列使用不同工具的分散步骤，每个步骤都需要专家级配置。目前实验室工作流如下：

| 步骤 | 工具 | 痛点 |
|------|------|------|
| 图像拼接 / Z 轴导出 | Fiji / 自定义脚本 | 每个数据集需手动调参 |
| 图谱切片匹配（AP 位置） | brainreg / 手动查表 | 需要对方向和半球做逐样本判断 |
| 切片到图谱配准 | brainreg / elastix / 手动 | 配准时没有可视反馈；不同工具输出格式不兼容 |
| 细胞 / 信号检测 | Cellpose / 手动阈值 | 模型选择和阈值需针对每次成像会话单独调整 |
| 区域映射 | cellfinder / 自定义代码 | 独立工具，输出格式与配准结果不兼容 |
| 聚合与统计 | Excel / R / Python 脚本 | 需手动合并来自 3-4 个不同工具的文件 |
| 可视化与质控 | Fiji / napari | 独立步骤，与定量结果无直接关联 |
| 报告 / 导出 | 手动 | Methods 靠记忆撰写，无参数可追溯性 |

**后果：** 一个脑样本需要切换 5-7 个工具、做 10-15 次手动参数决策，产出格式不兼容的文件。不同操作者结果存在差异。新成员需要数周时间学习整套工具链。复现之前的分析需要几乎不存在的文档。

### Brainfast 的解决方案

Brainfast **不是** brainreg、cellfinder 或 ClearMap 的替代品——这些工具各自在单一任务上表现很好。Brainfast 提供的是它们之间缺失的**连接层**：一个 GUI 驱动的单一 pipeline，输入原始 Z-stack，输出结构化的、图谱映射的、按区域定量的分析结果——全程参数可追溯，每步均有质控。

具体而言，Brainfast：

1. **将 7+ 个碎片化步骤压缩为 4 步 GUI 工作流**（配置 → 配准 → 审查 → 检测+导出）
2. **将 5-7 个分散工具替换**为一个本地桌面应用
3. **直接产出结构化生物学数据**——按区域细胞计数、95% CI、形态学指标、AP 轴密度分布——无需手动后处理
4. **将实验室经验编码为配置模板**（半球朝向、像素尺寸、增强子 AAV 光片数据通道映射）
5. **强制可重复性**——完整参数日志、图谱 SHA-256 指纹、自动生成 Methods 文本

这是 Brainfast 填补的空白：**将一个多工具、依赖专家、文档不完整的流程，转化为可重复的单入口分析 pipeline。**

## 贡献声明

本项目的贡献最准确地定义为 **pipeline 创新 / 分析基础设施贡献**，而非算法创新：

> *Brainfast 为增强子 AAV 光片数据建立了一条可重复、GUI 驱动的全脑分析流程，将 7 步、5 工具的碎片化工作流压缩为 4 步单应用流程，产出适合多样本比较分析的结构化按区域生物学数据。*

具体技术贡献：

- **统一的配准-检测-映射 pipeline**，单一配置文件，一个 GUI 入口
- **针对清脑半脑样本的实验室特定配准启发式算法**（right_flipped 模式、TPS 非线性变形针对组织边界贴合调优）
- **校准学习系统**（17 对人工标注训练数据），从操作者反馈中改进配准质量
- **结构化定量输出**（按区域计数、95% Garwood-Poisson CI、形态学列、AP 轴密度、跨样本透视表）
- **内置质控 pipeline**（逐切片 edge-SSIM、Z 轴连续性离群点检测、preflight 参数验证）
- **批处理与项目管理**，支持多脑可扩展分析

## 传统工作流 vs Brainfast

| 指标 | 传统流程（当前实验室）| Brainfast |
|------|---------------------|-----------|
| 所需工具数 | 5-7（Fiji、brainreg、cellfinder、Cellpose、Excel、R/Python、napari）| 1（Brainfast）|
| 每脑手动参数决策次数 | 10-15 | 3-5（像素尺寸、半球、通道）|
| 需要专家判断的步骤数 | 4-5 | 1-2（配准审查）|
| 参数可重复性 | 低（无文档）| 完整（config JSON + 图谱 SHA-256 + 自动 Methods）|
| 原始 Z-stack 到区域计数的时间 | 4-8 小时（有经验用户）| 1-2 小时（任意实验室成员）|
| 输出格式 | 不一致（因人而异）| 标准化 CSV + Excel，每次运行统一 |
| 新成员上手时间 | 2-4 周 | < 1 天（GUI + 引导游览）|
| 批处理支持 | 不支持 | FIFO 批处理队列，N 个脑无人值守运行 |

*运行时间估算基于 111 切片冠状半脑，5 µm/pixel。具体时间因 GPU 可用性和图像质量而异。*

## Pipeline 工作流

```
输入：TIFF Z-stack（例：111 个冠状切片，5 µm/pixel）
              │
              ▼
  ┌─────────────────────────┐
  │  第一步 · 配置           │  ← 单一 run_config.json（像素尺寸、半球朝向、
  │                         │    图谱路径、通道映射、检测参数）
  └────────────┬────────────┘
               │
               ▼
  ┌─────────────────────────┐
  │  第二步 · 配准           │  AP 切片匹配（Z 文件名 → 图谱坐标）
  │                         │  仿射初始放置 → TPS 非线性变形
  │                         │  输出：registered_slices/、slice_registration_qc.csv
  └────────────┬────────────┘
               │
               ▼
  ┌─────────────────────────┐
  │  第三步 · 审查 / 质控    │  ← 浏览器 GUI：逐切片叠加图、SSIM 分数、
  │                         │    手动液化校正（如需）、
  │                         │    Z 轴连续性 AP 轴图表
  └────────────┬────────────┘
               │
               ▼
  ┌─────────────────────────┐
  │  第四步 · 检测+导出      │  LoG 或 Cellpose 检测 → 3D KD-tree 去重
  │                         │  → CCFv3 区域映射 → 层次聚合
  │                         │  → Garwood 95% CI → Excel + CSV + Methods 文本
  └────────────┬────────────┘
               │
               ▼
输出：cell_counts_leaf.csv · cell_counts_hierarchy.csv
      cells_dedup.csv · brainfast_results.xlsx
      detection_summary.json（atlas_sha256、参数、总计数）
      自动生成 Methods 段落（EN + ZH）
```

## 生物学输出

Brainfast 的设计目标是回答这个问题：

> *"我的增强子 AAV 在大脑哪些区域表达，每个区域有多少细胞表达，跨样本之间如何比较？"*

每次 pipeline 运行产出：

| 输出 | 生物学意义 |
|------|-----------|
| `cell_counts_leaf.csv` | 叶级别按区域细胞计数，含 95% Poisson CI、平均伸长度、平均面积、平均强度 |
| `cell_counts_hierarchy.csv` | 按完整 Allen 本体树聚合（叶区 → 脑区 → 分区 → 主区）|
| `cells_dedup.csv` | 单个细胞质心（x, y, z_µm, score, region_id）——适合下游空间分析 |
| AP 轴密度图 | 每个前后轴坐标的细胞数——可视化 AAV 表达沿 AP 轴的集中分布 |
| 跨样本比较 | 透视表：行 = 脑区，列 = 样本/动物——支持 N > 1 的组间比较 |
| Methods 段落 | 自动生成可发表的可重复性文本，引用图谱版本、参数和 SHA-256 |

## 最小验证

> **注意：** Brainfast 已在一个试点脑样本（Sample 35，111 个冠状切片，5 µm/pixel，右半球，清脑组织）上完成验证。这是*最小概念验证*，不是系统性 benchmark。

**配准质量：** 每切片的 edge-SSIM 分数记录在 `slice_registration_qc.csv` 中。对配准叠加图的目视检查确认所有 111 个切片的半球朝向和解剖标志对齐正确。

**检测：** LoG 检测器输出在 3 个代表性 ROI 中与手动检查进行了对比。细胞质心位置与原始图像中肉眼可见的信号阳性细胞定性一致。

**当前验证的局限性：** 尚未进行正式的人工计数 vs 自动计数比对。详见[局限性](#局限性)章节。

## 快速开始

### 环境要求

- Python 3.10 或 3.11
- Windows 10/11（主要平台）· Linux 通过 Docker
- `annotation_25.nii.gz` — Allen CCFv3 25 µm 注释文件（放置于 `project/`）
- 建议 NVIDIA GPU（用于 Cellpose）；LoG 检测器可在 CPU 上运行

### 安装

```powershell
git clone https://github.com/Skiyoshika/Brainfast.git
cd Brainfast
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -e ".[advanced,dev]"
```

### 运行

```powershell
.\Start_Brainfast.bat
# 或：python project\frontend\server.py
```

在浏览器中打开 **http://127.0.0.1:8787**。

## 局限性

1. **仅单脑验证。** 所有参数在 Sample 35 上调优，尚未在其他增强子、成像条件或脑区上系统评估。
2. **无正式准确性 benchmark。** 需要系统的 ROI 级人工计数 vs 自动计数比对。
3. **LoG 检测器局限性。** 在密集区域或低信噪比图像中可能存在过/漏计数。
4. **配准质量依赖数据。** 全脑、矢状切片或不同分辨率需重新调参。
5. **无动物间归一化。** 细胞数为原始计数，脑体积归一化需在下游进行。
6. **Windows 主平台。** Linux 支持通过 Docker（无头模式）。
